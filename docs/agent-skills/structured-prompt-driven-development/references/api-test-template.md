1. **`[ID]-[YYYYMMDDHHMM]-[ApiTest]-Title.md`** — human-readable scenario table.
2. **`scripts/spdd/test-<canvasId>.sh`** — runnable bash with cURL, exit 0 on all-pass.

## Scenarios

### scenario-001 — within-quota (normal, op-001)

**Given:**
- Customer `cust-1` on plan `Standard`, quota 100_000, used 90_000 this period.
- Pricing: fast-model overage = $0.01/1K.

**Setup:**
```bash
curl -X POST $BASE/customers \
  -d '{"id":"cust-1","plan":"Standard"}' \
  -H 'Content-Type: application/json'
curl -X POST $BASE/internal/seed-usage \
  -d '{"customerId":"cust-1","tokens":90000,"model":"fast-model"}' \
  -H 'Content-Type: application/json'
```

**When:**
```bash
curl -X POST $BASE/usage \
  -d '{"customerId":"cust-1","tokens":30000,"model":"fast-model","eventId":"evt-1"}' \
  -H 'Content-Type: application/json' \
  -o /tmp/scenario-001.json -w '%{http_code}\n'
```

**Then:**
- HTTP status: `200`
- Response JSON shape:
  ```json
  {
    "charged": "0.20",
    "fromQuota": 10000,
    "overage": 20000,
    "currency": "USD"
  }
  ```
- Validation:
  ```bash
  jq -e '.charged == "0.20" and .fromQuota == 10000 and .overage == 20000' /tmp/scenario-001.json
  ```

### scenario-003 — unknown-model (error, op-001)

**Given:** customer exists, plan has no rule for `weird-model`.

**When:**
```bash
curl -X POST $BASE/usage \
  -d '{"customerId":"cust-1","tokens":1,"model":"weird-model","eventId":"evt-2"}' \
  -o /tmp/scenario-003.json -w '%{http_code}\n'
```

**Then:**
- HTTP status: `400`
- Response shape: `{"error": {"code": "UNKNOWN_MODEL", "model": "weird-model"}}`
- No new ledger row exists for `evt-2`:
  ```bash
  curl $BASE/internal/ledger/cust-1 | jq -e '[.entries[] | select(.eventId == "evt-2")] | length == 0'
  ```
```

## Script template

```bash
#!/usr/bin/env bash
# Generated by /spdd-api-test for canvas <canvasId>
# Run: ./scripts/spdd/test-<canvasId>.sh
# Exits 0 only if every assertion passes.

set -u
BASE="${BASE:-http://localhost:8080}"
PASS=0
FAIL=0
trap 'echo "Aborted"; exit 2' INT

run() {
  local name=$1; shift
  if "$@"; then
    echo "  ✅ $name"
    PASS=$((PASS+1))
  else
    echo "  ❌ $name"
    FAIL=$((FAIL+1))
  fi
}

echo "scenario-001 within-quota"
# ... setup, when, then ...
run "scenario-001 charged=0.20" \
  jq -e '.charged == "0.20" and .fromQuota == 10000 and .overage == 20000' /tmp/scenario-001.json

echo
echo "Passed: $PASS, Failed: $FAIL"
test "$FAIL" -eq 0
```

## Rules

- **Idempotent**: re-running the script produces the same pass/fail result. Use deterministic IDs.
- **Self-cleaning**: each scenario sets up its own state and tears down (or uses a fresh test workspace).
- **Specific**: every assertion checks a concrete value, never just `200 OK`.
- **Numbered**: scenarios `scenario-NNN` so they map back to the AC they prove.

## Pass criteria for the gate

- Every Operation in the Canvas appears in the coverage matrix.
- The `.sh` script exits 0 against the running service.
- Each scenario references at least one AC.
- Re-runs are deterministic (no flakes).
